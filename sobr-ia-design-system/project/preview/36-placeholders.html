⑥ Contrat d'usage · à respecter dans tous les composants

Toute carte ou dataviz du produit doit implémenter ces 5 props.

5 props obligatoires sur chaque composant à données :

  • state : loading | empty | partial | error | ready
  • onRetry : callback affiché en cas d'erreur réseau
  • emptyAction : {label, href, primary} — CTA pour amorcer la data
  • fallbackUnit : ce qu'on affiche à la place de la valeur (ex. "—", "∅", "~ estimé")
  • explainWhy : phrase d'1 ligne expliquant pourquoi rien à montrer

Règle d'or : jamais un graphe vide, jamais une jauge à 0 sans contexte, jamais un "—" sans phrase d'accompagnement.
