# C33.5 — Workflows release CI + section #download + 3 pages secondaires — Plan

> REQUIRED SUB-SKILL: superpowers:executing-plans.

**Goal:** Rendre les téléchargements proposés sur la landing réels (workflows GitHub Actions qui buildent les binaires Tauri + l'extension), créer la section `#download` avec auto-détection OS, et livrer les 3 pages secondaires (/telecharger/, /cloud/, /candidature/) — sans toucher à GitHub releases existantes depuis ce poste (le retro-trigger v0.7.1 sera fait par Thibault depuis l'UI GitHub Actions une fois les workflows mergés).

**Architecture & décisions:**
- Workflows triggés sur **tag push `v*.*.*`** + **workflow_dispatch** (flexibilité retro).
- macOS = 2 jobs séparés (x86_64 sur runner Intel, aarch64 sur runner ARM via `runs-on: macos-latest` qui est ARM en 2025). `macos-13` reste sur Intel.
- Pas de codesign en v0.1.0 (différé) — la page `/telecharger/` documente les avertissements Gatekeeper macOS et SmartScreen Windows.
- DownloadSection détecte OS via `navigator.userAgent` + `navigator.userAgentData.platform`, highlight la carte recommandée.
- URLs GitHub stables : `https://github.com/BkOff-fr/defis-lia-generatif/releases/latest/download/<filename>`.
- Form `/cloud/` = `mailto:contact@sobria.brilliantstudio.co?subject=...` (RGPD-friendly, zéro service tiers).

**Filenames Tauri attendus** (correspondent à `bundle.targets: "all"` + `productName: "Sobr.ia"`, version 0.8.0 actuelle) :
- Windows : `Sobr.ia_<version>_x64-setup.exe` (NSIS) + `Sobr.ia_<version>_x64_en-US.msi`
- macOS Intel : `Sobr.ia_<version>_x64.dmg`
- macOS ARM : `Sobr.ia_<version>_aarch64.dmg`
- Linux : `sobr-ia_<version>_amd64.deb` + `Sobr.ia_<version>_amd64.AppImage`

**Filenames extension** :
- `sobria-extension-chrome-v<version>.zip`
- `sobria-extension-firefox-v<version>.xpi`
